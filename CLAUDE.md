# CLAUDE.md — 健康Tips自動収集・発信プロジェクト

## プロジェクト概要

理学療法士である太一（ペンネーム：すえ）が、自分と周囲の人の健康を守るために、
毎日1つの健康Tipsを収集・整理・発信する自動化パイプライン。

**コンセプト：「AIが調べる、PTが伝える」**
AIが情報収集と下書きを担当し、理学療法士としての視点でひとことを添えて発信する。

## 情報収集の方針

### 収集ソース（バランス重視）

| 種別 | ソース | 備考 |
|------|--------|------|
| 学術・公的機関 | PubMed API | 英語論文→日本語要約 |
| 学術・公的機関 | 厚生労働省 e-ヘルスネット | RSSまたはスクレイピング |
| 学術・公的機関 | 国立健康・栄養研究所 | 信頼性高い |
| 一般メディア | NHK健康チャンネル | RSS取得 |
| 一般メディア | ヘルスケア大学 | RSS取得 |

### 収集のルール

- 1日1テーマ、1Tips に絞る（多すぎない）
- テーマカテゴリをローテーション：
  - 月：食事・栄養
  - 火：運動・リハビリ
  - 水：睡眠・休養
  - 木：メンタル・ストレス
  - 金：予防・生活習慣
  - 土：骨・関節・筋肉（PT専門領域）
  - 日：自由テーマ（旬の話題）

### システムプロンプト

```
あなたは理学療法士「すえ」のアシスタントです。
収集した健康情報を、以下のフォーマットで整形してください。

【発信対象】リベシティコミュニティ、家族・友人など身近な人
【トーン】親しみやすく、専門用語は使わず、でも根拠あり
【文体】です・ます調。断定しすぎず、「〜と言われています」「〜が大切です」
```

### 出力フォーマット

```markdown

{すぐコピペして投稿できるテキスト。ハッシュタグ含む}

**信頼度**：★★★（論文・公的機関）/ ★★（信頼メディア）/ ★（参考程度）
```

## 「PTすえのひとこと」生成ルール

Claude Codeが自動生成するひとことは、以下の知識ベースを参照すること：

- 介護・デイケア現場での実体験を反映する
- 「転倒予防」「廃用予防」「QOL」の視点を優先
- 患者・利用者への声かけに使える言葉にする
- 自分の習慣（犬の散歩・筋トレ・ストレッチ）と絡められるなら絡める
- 太一自身のパートナー・家族への思いを根底に持つ

## 発信フロー

```
1. collect.py 実行（毎朝7:00 cronで自動化）
      ↓
2. summarize.py 実行（Claude APIで3本分を整形・JSON保存）
      ↓
3. output/daily/YYYY-MM-DD.json に保存
      ↓
4. build_html.py 実行（JSONをnote風HTMLに変換）
      ↓
5. output/html/YYYY-MM-DD.html に保存
      ↓
6. LINE Messaging APIで自分のLINEに通知（3本のタイトル＋要点をテキスト送信）
      ↓
7. 太一がHTMLを開いて3本を読む・「PTひとこと」を必要なら手直し
      ↓
8. リベシティ or 身近な人に投稿用テキストをコピペしてシェア
```

## 週次まとめ（Phase 2以降）

毎週日曜夜に `export.py` が以下を自動生成：

- 週7本のTipsをカテゴリ別に整理
- 新しいWordPressブログへの投稿下書き（HTML or Markdown）
- タイトル候補3案をClaude APIで生成

## 環境変数（.env）

```
# LINE Messaging API（自分のLINEに通知）
LINE_CHANNEL_ACCESS_TOKEN=your_token_here
LINE_USER_ID=Uxxxxxxxxxxxxxxxxxx

# PubMed API（任意：なくても動作可）
PUBMED_API_KEY=your_key_here
```

### 各キーの取得方法

**LINE_CHANNEL_ACCESS_TOKEN と LINE_USER_ID**
1. [developers.line.biz](https://developers.line.biz) にLINEアカウントでログイン
2. 「プロバイダー作成」→「Messaging APIチャンネル」を作成
3. 「Messaging API」タブ→「チャンネルアクセストークン」を発行
4. 「チャンネル基本設定」タブ→「あなたのユーザーID」をコピー
5. チャンネルのQRコードを自分のLINEで友だち追加（友だち追加しないと通知が届かない）

**PUBMED_API_KEY**
1. [ncbi.nlm.nih.gov](https://www.ncbi.nlm.nih.gov/account/) でアカウント作成
2. 「API Key Management」→「Create an API Key」

> ⚠️ `.env` はGit管理外にすること。Claude Codeに「.gitignoreに.envを追加して」と指示すればOK。

## UIテンプレート仕様（viewer.html）

`templates/viewer.html` はnoteライクなデザインのHTMLテンプレート。
**このファイルはデザインの基準となるため、直接編集しないこと。**
`build_html.py` がこのテンプレートを読み込み、Tipsデータを流し込んでHTMLを生成する。

### テンプレートの構造

```
ヘッダー（ロゴ・日付）
タブナビ（今日の3本 / 今週のまとめ / アーカイブ）
  └── 一覧ビュー：3枚のカード（カテゴリ・タイトル・リード文・信頼度）
        ↓ タップ
  └── 詳細ビュー（1本ずつ）
        ├── カテゴリバッジ・タイトル
        ├── 著者行（すえ / 理学療法士 / Fukui）
        ├── 今日のTips（緑ボーダーカード）
        ├── もう少し詳しく
        ├── PTすえのひとこと（吹き出し風カード）
        ├── リベシティ投稿用テキスト（コピーボタン付き）
        ├── 情報源・信頼度★表示
        └── タグ・前後Tips ナビ
```

### 1日3本のテーマ構成

毎日3テーマを固定ローテーションで組み合わせる：

| 曜日 | 1本目 | 2本目 | 3本目 |
|------|-------|-------|-------|
| 月 | 食事・栄養 | 運動・リハビリ | メンタル |
| 火 | 運動・リハビリ | 食事・栄養 | 睡眠 |
| 水 | 睡眠・休養 | 予防・生活習慣 | 食事・栄養 |
| 木 | メンタル・ストレス | 睡眠 | 運動 |
| 金 | 予防・生活習慣 | 骨・関節・筋肉 | 食事 |
| 土 | 骨・関節・筋肉 | 運動 | メンタル |
| 日 | 自由テーマ | 食事 | 睡眠 |

### JSONデータ仕様（output/daily/YYYY-MM-DD.json）

`build_html.py` が読み込む中間データ形式：

```json
{
  "date": "2026-04-15",
  "tips": [
    {
      "category": "🏃 運動・リハビリ",
      "title": "記事タイトル",
      "core": "今日のTips本文（1〜2文）",
      "detail": "詳細本文（3〜5文）",
      "pt": "PTすえのひとこと（150字以内）",
      "post": "リベシティ投稿用テキスト（ハッシュタグ含む）",
      "source": "情報源名",
      "stars": "★★★",
      "reliability": "高（査読論文）",
      "tags": ["#タグ1", "#タグ2"]
    }
  ]
}
```

## Claude Codeへの作業指示の出し方

### 初回セットアップ時

```
「collect.pyを作成してください。sources.yamlに従いNHK健康チャンネルRSSとPubMed APIからTipsを収集し、rawデータをoutput/daily/に保存してください」
```

```
「build_html.pyを作成してください。output/daily/YYYY-MM-DD.jsonを読み込み、templates/viewer.htmlのテンプレートにデータを流し込んでoutput/html/YYYY-MM-DD.htmlを生成してください」
```

### 日常メンテナンス時

```
「今日のTips（YYYY-MM-DD.json）のPTひとことを書き直してください。今日は転倒予防についてのTipsなので、デイケア現場の視点を入れてください」
```

```
「build_html.pyを実行して今日のHTMLを再生成してください」
```

### 新しいソース追加時

```
「sources.yamlに『〇〇』のRSSを追加し、collect.pyがそこからも取得できるようにしてください」
```

### UIを調整したい時

```
「templates/viewer.htmlのカードデザインを調整してください。カード背景色を〇〇に変えてください」
```

## 禁止事項・注意事項

- 医療アドバイス・診断に当たる表現は避ける（「〜してください」より「〜が大切と言われています」）
- 根拠不明な民間療法・サプリ情報は収集しない
- 信頼度★のソースは必ず「参考情報」として注記する
- 個人を特定できる患者情報は絶対に含めない
